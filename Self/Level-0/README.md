## ðŸ”‘ Key Patterns to Master

### 1. Two Pointer Technique
- **Used in:** Arrays, Strings, Linked List
- **Examples:** Two Sum (sorted), Container with Most Water, Palindrome check
- **When to use:** When you need to compare/process elements from both ends or maintain a window

### 2. Sliding Window
- **Used in:** Arrays, Strings
- **Examples:** Longest Substring, Maximum Sum Subarray, Anagrams in String
- **When to use:** Finding subarrays/substrings with specific properties

### 3. Frequency Counter
- **Used in:** Arrays, Strings, HashMap
- **Examples:** Anagrams, First Unique Character, Top K Frequent
- **When to use:** Counting occurrences, finding duplicates/unique elements

### 4. Prefix Sum
- **Used in:** Arrays, HashMap
- **Examples:** Subarray Sum Equals K, Range Sum Query
- **When to use:** Range queries, subarray sum problems

### 5. Fast-Slow Pointer (Floyd's Algorithm)
- **Used in:** Linked List
- **Examples:** Detect Cycle, Find Middle, Palindrome Check
- **When to use:** Cycle detection, finding middle element

### 6. Monotonic Stack/Deque
- **Used in:** Stack, Queue
- **Examples:** Next Greater Element, Sliding Window Maximum
- **When to use:** Finding next/previous greater/smaller elements

### 7. BFS with Queue
- **Used in:** Queue, Trees, Graphs
- **Examples:** Level-order Traversal, Shortest Path, Rotting Oranges
- **When to use:** Level-by-level processing, shortest path problems

### 8. HashMap for Complement/Pairing
- **Used in:** HashMap, Arrays
- **Examples:** Two Sum, Subarray Sum, Group Anagrams
- **When to use:** Finding pairs, tracking seen elements

## ðŸ’¡ Problem-Solving Strategy

### 1. Understand the Problem
- Read carefully and identify inputs/outputs
- Check constraints (array size, value ranges)
- Identify edge cases

### 2. Pattern Recognition
- Which data structure is most suitable?
- Have you seen a similar problem?
- What pattern does this follow?

### 3. Approach Selection
- Start with brute force (explain it)
- Optimize using appropriate pattern
- Consider time/space complexity

### 4. Implementation
- Use meaningful variable names
- Handle edge cases
- Test with examples

### 5. Optimization
- Can you reduce time complexity?
- Can you save space?
- Are there trade-offs?
